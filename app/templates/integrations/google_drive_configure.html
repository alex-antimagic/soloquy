Setup Instructions
1. Create Google Cloud Project

Go to Google Cloud Console and create a new project (or use existing)

2. Enable Google Drive API

In "APIs & Services" → "Library", search for "Google Drive API" and enable it

3. Create OAuth Credentials

In "APIs & Services" → "Credentials":

  • Click "Create Credentials" → "OAuth client ID"
  • Application type: "Web application"
  • Add this redirect URI:
    {{ url_for('integrations.google_drive_callback', scope=scope, _external=True) }}
4. Configure OAuth Consent Screen

{% if is_workspace %} Choose "Internal" if using Google Workspace, or "External" for personal accounts {% else %} Choose "External" and add yourself as a test user {% endif %}

5. Required Scopes
  • drive (Full access to Drive)
  • drive.file (Per-file access)
Note: You can use the same Google Cloud project for both Gmail and Drive. Just enable both APIs and add the necessary scopes.
Security Note: Your credentials are encrypted at rest. The Drive MCP server runs as a separate process and only your authorized agents can access it.
